# 是啊，吃什么

![插件图标](logo.png)

一款为 AstrBot 开发的简单有趣的插件，帮助大学生解决每天最头疼的问题——今天吃什么？

## 🎯 功能特点

- **智能识别**：自动识别消息中的"吃什么"关键词
- **自定义关键词**：支持配置多个触发关键词（v1.3.0新增）
- **食物配图**：为食物绑定图片，推荐时图文一起发送（v1.3.0新增）
- **随机推荐**：根据设置的概率随机推荐美食或复读"是啊，吃什么"
- **可编辑食物库**：内置50个大学生常吃的食物，支持自由增删改
- **自定义扩展**：可添加额外的自定义食物列表
- **概率配置**：灵活调整推荐 vs 复读的概率比例
- **频率限制**：防止多Bot循环触发（v1.1.0新增）
- **复读冷却**：复读后的一段时间内强制推荐食物，避免过度复读（v1.2.0新增）
- **参数兼容**：适配 AstrBot 多种 handler 调用方式（v1.1.1修复）

## 📦 安装方法

### 方法一：WebUI 上传（推荐）

1. 打开 AstrBot 的 WebUI
2. 进入"插件市场"
3. 点击"上传插件"
4. 选择 `astrbot_plugin_what_to_eat.zip` 文件
5. 重启 AstrBot

### 方法二：手动安装

1. 将插件 ZIP 文件解压到 `data/plugins/astrbot_plugin_what_to_eat/`
2. 重启 AstrBot
3. 在 WebUI 中启用插件

## ⚙️ 配置说明

在 WebUI → 插件管理 → 是啊，吃什么 → 配置 中可以进行以下设置：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `trigger_keywords` | list | ["吃什么"] | 触发插件的关键词列表（v1.3.0） |
| `uploaded_images` | file | [] | 上传的食物图片（v1.3.0） |
| `recommend_probability` | float | 0.3 | 推荐食物的概率（0.0-1.0） |
| `builtin_foods` | list | 50个食物 | 内置食物列表，可自由编辑 |
| `custom_foods` | list | [] | 额外的自定义食物列表 |
| `rate_limit_enabled` | bool | true | 启用频率限制（防多Bot循环） |
| `rate_limit_max` | int | 3 | 每分钟最大响应次数 |
| `echo_cooldown_enabled` | bool | true | 启用复读冷却功能 |
| `echo_cooldown_seconds` | int | 15 | 复读后的冷却时间（秒） |

### 频率限制说明（v1.1.0）

**问题场景**：群内有两个Bot都安装了此插件，当一个Bot回复"吃什么"时，另一个Bot也会触发，导致无限循环。

**解决方案**：
- 启用频率限制后，插件会记录每分钟的响应次数
- 超过 `rate_limit_max` 次（默认3次）后，后续响应会100%推荐食物（而非复读）
- 时间窗口为1分钟，超时后计数自动重置
- 不同群组互不影响

### 复读冷却说明（v1.2.0）

**问题场景**：当Bot复读"是啊，吃什么"后，如果用户继续发送"吃什么"，Bot可能会连续复读，显得不智能。

**解决方案**：
- 启用复读冷却后，当Bot复读"是啊，吃什么"后，会进入冷却期
- 冷却期内（默认15秒）如果再触发，Bot会强制推荐食物而不是继续复读
- 冷却时间结束后，恢复正常概率
- 与频率限制独立工作，可同时启用

### 自定义关键词（v1.3.0）

**功能**：支持配置多个触发关键词，不限于"吃什么"。

**配置示例**：
```json
{
  "trigger_keywords": ["吃什么", "吃啥", "吃点什么", "今天吃啥"]
}
```

**说明**：
- 配置多个关键词后，任一关键词触发都会响应
- 关键词列表为空时自动使用默认 `["吃什么"]`
- 关键词支持中文、英文、符号等

### 食物配图（v1.3.0）

**功能**：为食物添加图片，推荐时图文一起发送。

**两种方式添加图片**：

#### 方式一：WebUI 直接上传（推荐）
1. 在插件配置的 `uploaded_images` 处上传图片
2. 文件名格式：`食物名.jpg` 或 `食物名_1.jpg`
3. 上传后自动绑定到对应食物

#### 方式二：手动放入文件夹
1. 在插件目录下创建 `images/` 文件夹
2. 把图片放入该文件夹
3. 重启 AstrBot 生效

**文件名格式**：
- `黄焖鸡米饭.jpg` - 基础格式
- `黄焖鸡米饭_1.jpg` - 带序号格式（支持多图随机）
- `麻辣烫.png` - 其他格式也支持

**说明**：
- 支持的格式：jpg、jpeg、png、gif、webp、bmp
- 一个食物可放多张图片，随机选择一张发送
- 两种方式可同时使用，图片会合并
- 无图片时正常发送文字推荐
- 复读时不发送图片，保持简洁

### 配置示例

**推荐概率设为 0.5（一半概率推荐）：**


**关闭频率限制（不推荐）：**
```json

  "rate_limit_enabled": false
}
```

**调整频率限制次数：**
```json
{
  "rate_limit_max": 5

```

**调整复读冷却时间：**
```json
{
  "echo_cooldown_seconds": 10
}
```

**关闭复读冷却：**
```json
{
  "echo_cooldown_enabled": false
}
```

## 💬 使用示例

### 基本用法

用户发送包含"吃什么"的消息：

```
用户：今天中午吃什么好呢？
机器人：要不吃黄焖鸡米饭？

用户：晚上吃什么
机器人：是啊，吃什么

用户：@机器人 吃什么
机器人：试试麻辣烫吧！
```

### 频率限制示例（v1.1.0）

```
用户：吃什么？          ← 机器人A：是啊，吃什么（第1次）
用户：吃什么？          ← 机器人B：是啊，吃什么（第1次）
用户：吃什么？          ← 机器人A：是啊，吃什么（第2次）
用户：吃什么？          ← 机器人B：是啊，吃什么（第2次）
用户：吃什么？          ← 机器人A：是啊，吃什么（第3次）
用户：吃什么？          ← 机器人B：是啊，吃什么（第3次）
用户：吃什么？          ← 机器人A：要不吃黄焖鸡米饭？（超限，强制推荐）
用户：吃什么？          ← 机器人B：要不吃麻辣烫？（超限，强制推荐）
```

### 复读冷却示例（v1.2.0）

```
用户：吃什么？          ← Bot：是啊，吃什么（复读，进入15秒冷却）
用户：吃什么？          ← Bot：要不吃黄焖鸡米饭？（冷却中，强制推荐）
用户：吃什么？          ← Bot：试试麻辣烫吧！（冷却中，强制推荐）
...等待15秒后...
用户：吃什么？          ← Bot：是啊，吃什么（冷却结束，可以复读了）
```

## 🎲 概率说明

- **推荐食物**（概率：P）：从食物列表中随机选择一个推荐
- **复读**（概率：1-P）：回复"是啊，吃什么"
- **强制推荐**：频率限制超限时，100%推荐食物

默认概率为 0.3，即：
- 30% 概率推荐食物
- 70% 概率复读"是啊，吃什么"

## 🔧 技术细节

- **触发方式**：正则表达式匹配 `吃什么`
- **不需要@**：直接发送包含关键词的消息即可触发
- **阻断LLM**：识别到关键词后自动阻止默认LLM回复
- **频率限制**：按群组记录响应次数，防止多Bot循环
- **复读冷却**：复读后的一段时间内强制推荐食物，避免过度复读
- **群组/私聊**：都支持
- **线程安全**：使用随机数生成器安全选择食物

## 🐛 故障排除

### 插件加载失败
1. 检查 ZIP 文件是否完整
2. 确认 AstrBot 版本支持
3. 查看日志中的错误信息

### 不触发回复
1. 确认插件已启用
2. 检查消息是否包含"吃什么"三个字
3. 查看是否有其他插件冲突

### 食物列表为空
- 确保 `builtin_foods` 或 `custom_foods` 至少有一个不为空
- 检查配置文件格式是否正确

### LLM仍然回复
- 确保插件已正确安装和启用
- 检查是否有其他插件干扰

### 频率限制不生效
- 确认 `rate_limit_enabled` 为 `true`
- 检查 `rate_limit_max` 设置是否合理

### 复读冷却不生效
- 确认 `echo_cooldown_enabled` 为 `true`
- 检查 `echo_cooldown_seconds` 设置是否合理（建议5-30秒）
- 注意：只有复读"是啊，吃什么"后才会进入冷却，推荐食物不会触发
