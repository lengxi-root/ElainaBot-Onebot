# astrbot_plugin_dnd_dice

适用于 [AstrBot](https://github.com/AstrBotDevs/AstrBot) 的 D&D 骰子插件，实现部分 Roll20 骰池规范，支持技能检定判定及 LLM
函数工具自动调用。

## 功能

- **基础骰池**：`d20`、`2d6+5`、`4d6kh3` 等完整 DnD 语法
- **FATE/Fudge 骰**：`4dF`（-1 / 0 / +1 三面骰）
- **保留骰**：`4d6kh3`（取高 3）、`2d20kl1`（取低 1）、`k` 作为 `kh` 简写
- **丢弃骰**：`8d6d3` / `8d6dl3`（丢弃最低 3）、`8d6dh3`（丢弃最高 3）
- **优势 / 劣势**：`d20adv` / `d20dis`
- **标准爆炸骰**：`d6!`、自定义阈值 `d6!>4`（掷出 ≥4 即爆炸）
- **复合爆炸骰**：`5d6!!`（Shadowrun 风格，追加值合并为单骰）
- **穿透爆炸骰**：`5d6!p`（HackMaster 风格，追加骰各减 1）
- **目标数成功计数**：`3d6>3`（≥3 算成功）、`10d6<4`（≤4 算成功）
- **失败计数**：`3d6>3f1`（1 算失败，从成功数中扣除）
- **重骰**：`2d8r<2`（循环重掷直至满足条件）
- **单次重骰**：`2d6ro<2`（每个骰子最多重掷一次）
- **结果排序**：`8d6s`（升序）、`8d6sd`（降序）
- **多骰组合**：`2d6+1d4+3`、`1d8-1d6`
- **标签注释**：`d20 攻击检定`、`d20#攻击检定`
- **DC 技能检定**：`d20 感知 15` → 输出成功 / 失败
- **天然 20 / 天然 1**：大成功 / 大失败特殊提示
- **LLM 函数工具**：LLM 可在 TRPG 叙事中自动调用掷骰

## 安装

在 AstrBot 管理面板的插件市场中搜索 `astrbot_plugin_dnd_dice` 并安装，或将本仓库克隆至 `data/plugins/` 目录下。

## 指令

| 指令               | 说明                            |
|------------------|-------------------------------|
| `/r`             | 掷一个 dN（N 为当前会话默认骰面数，初始为 d20）  |
| `/r [表达式]`       | 按指定骰池表达式掷骰                    |
| `/roll [表达式]`    | `/r` 的别名                      |
| `/dset`          | 查询当前会话的默认骰面数                  |
| `/dset <面数>`     | 将当前会话的默认骰面数设为指定值（如 `/dset 6`） |
| `/dset reset`    | 清除会话设置，恢复为全局默认骰面数             |
| `/dice_set [参数]` | `/dset` 的别名                   |
| `/rprefix`       | 查询当前会话的自定义触发前缀                |
| `/rprefix <符号>`  | 将当前会话触发前缀设为指定符号（如 `.` 或 `!`）  |
| `/rprefix reset` | 清除会话前缀，恢复全局默认                 |
| `/rh`            | 查看当前会话投掷历史（群聊显示全员，私聊显示本人）     |
| `/rh me`         | 仅显示自己的投掷记录（群聊内使用）             |
| `/rh clear`      | 清空当前会话历史（群聊需白名单权限）            |
| `/rhistory [参数]` | `/rh` 的别名                     |

## 骰面大小设置

通过 `/dset` 指令可以为当前会话（群聊或私聊）独立设置默认骰面数，使得 `/r` 无参数时投掷指定面数的骰子。

```
/dset          # 查询当前会话默认骰面数及来源（会话设置 / 全局默认）
/dset 6        # 将本会话默认改为 d6
/dset 100      # 将本会话默认改为 d100
/dset reset    # 清除会话设置，恢复全局默认
```

设置后，`/r` 等价于 `/r d{N}`：

| 场景                    | 指令       | 等价于      |
|-----------------------|----------|----------|
| 全局默认 d20（未设置）         | `/r`     | `/r d20` |
| 会话设为 d6 后             | `/r`     | `/r d6`  |
| 会话设为 d6，但明确传入表达式时不受影响 | `/r d20` | `/r d20` |

### 权限控制

管理员可通过以下配置项控制 `/dset` 的使用权限：

- **`allow_session_dice_sides`**（默认 `true`）：设为 `false` 后，所有用户均无法使用 `/dset`。
- **`enable_whitelist`**（默认 `false`）：启用后，仅白名单用户（或 AstrBot 管理员）可使用 `/dset` 和 `/rprefix` 等**管理命令**
  。注意：此白名单不影响 `/r` 掷骰指令，掷骰对所有用户始终开放。
- **`whitelist_users`**：白名单用户 ID 列表，留空则使用 AstrBot 全局管理员 ID 作为白名单。
- **`allow_private_bypass_whitelist`**（默认 `true`）：启用后，私聊会话中的用户无需在白名单中即可使用 `/dset`。

## 投掷历史记录

插件会将每次成功投掷记录到当前会话（群聊或私聊）的历史中，可通过 `/rh` 指令查看或清空。

```
/rh            # 查看会话全部历史（群聊显示所有成员的投掷，私聊仅显示本人）
/rh all        # 同无参数
/rh me         # 仅显示自己的投掷记录（群聊内使用）
/rh clear      # 清空当前会话所有历史（群聊需白名单权限）
```

历史记录为会话级，群聊和私聊各自独立。每条记录包含：

| 字段  | 说明                  |
|-----|---------------------|
| 时间  | `MM-DD HH:MM:SS` 格式 |
| 表达式 | 原始骰池表达式             |
| 结果  | 投掷结果首行（超长自动截断）      |
| 发送者 | 群聊模式下显示昵称，私聊模式不显示   |

单次查询最多显示 20 条最新记录；历史条数上限可在配置项 `max_history_count` 中调整。失败投掷（解析错误、骰面超限等）不会写入历史。

### 权限控制

- **`enable_history`**（默认 `true`）：设为 `false` 后，插件不再记录任何历史，`/rh` 也不会生效。
- **`allow_view_history`**（默认 `true`）：即使历史记录已开启，设为 `false` 后用户无法通过 `/rh` 查看历史。
- 群聊中执行 `/rh clear` 需通过白名单验证；私聊任何人均可清除自己的历史。

## 自定义触发前缀

默认情况下，插件响应 AstrBot 系统级指令前缀（通常为 `/`），即 `/r`、`/roll`、`/dset` 等。  
通过自定义触发前缀功能，可以额外让插件响应其他符号（如 `.`、`!` 等），便于在不同平台或使用习惯下触发骰子指令。

> **注意**：自定义前缀是**额外**追加的，不会替换系统默认前缀。设置 `.` 后，`/r` 和 `.r` 都可以触发掷骰。

### 全局默认前缀

在管理面板配置 `default_cmd_prefix`（如填写 `.`），则所有会话均额外响应该前缀，无需用户单独设置。

### 会话级前缀

用户可通过 `/rprefix` 指令为当前会话设置独立前缀，优先级高于全局配置：

```
/rprefix           # 查询当前会话的有效前缀
/rprefix .         # 将本会话前缀设为 .
/rprefix !         # 将本会话前缀设为 !
/rprefix reset     # 清除会话前缀，恢复全局配置
```

设置后，对应前缀的所有掷骰指令均可使用：

| 场景           | 可用指令示例                           |
|--------------|----------------------------------|
| 系统默认（无自定义前缀） | `/r 1d20`、`/dset 6`              |
| 设置前缀 `.` 后   | `.r 1d20`、`.dset 6`（同时保留 `/r` 等） |
| 设置会话前缀 `!` 后 | `!r 2d6kh1`、`!roll d20adv`       |

### 权限控制

管理员可通过以下配置项控制 `/rprefix` 的使用权限：

- **`allow_custom_prefix`**（默认 `true`）：设为 `false` 后，所有用户均无法使用 `/rprefix` 设置前缀。
- **`enable_whitelist`**、**`whitelist_users`**、**`allow_private_bypass_whitelist`**：与 `/dset` 共用同一套白名单配置。

## 语法参考

### 基础

| 输入                | 输出示例                               |
|-------------------|------------------------------------|
| `/r d20`          | `d20: [14] = 14`                   |
| `/r 1d20+5`       | `d20+5: [9] +5 = 14`               |
| `/r 2d6-1`        | `2d6-1: [4, 3] -1 = 6`             |
| `/r 4dF`          | `4dF: [+, 0, -, +] = 1`            |
| `/r 2d6+1d4+3 伤害` | `伤害 2d6+1d4+3: [3, 5] [2] +3 = 13` |
| `/r 1d20+5#攻击检定`  | `攻击检定 d20+5: [13] +5 = 18`         |

### 保留 / 丢弃

| 输入           | 说明            | 输出示例                                          |
|--------------|---------------|-----------------------------------------------|
| `/r 4d6kh3`  | 取最高 3         | `4d6kh3: [(2), 4, 5, 6] = 15`                 |
| `/r 8d100k4` | `k` = `kh` 简写 | `8d100kh4: [(...), 76, 90, 98, 99] = 363`     |
| `/r 2d20kl1` | 取最低 1（劣势）     | `2d20kl1: [(18), 7] = 7`                      |
| `/r d20adv`  | 优势（2d20kh1）   | `2d20adv: [(8), 17] = 17`                     |
| `/r d20dis`  | 劣势（2d20kl1）   | `2d20dis: [11, (15)] = 11`                    |
| `/r 8d6d3`   | 丢弃最低 3        | `8d6dl3: [(1), (1), (2), 3, 4, 5, 5, 6] = 23` |
| `/r 8d6dh3`  | 丢弃最高 3        | `8d6dh3: [1, 2, 3, 4, (5), (6), (6)] = 10`    |

### 爆炸骰

| 输入         | 说明            | 输出示例                              |
|------------|---------------|-----------------------------------|
| `/r d6!`   | 标准爆炸（掷出 6 追加） | `d6!: [6, 4!] = 10`               |
| `/r d6!>4` | 掷出 ≥4 即爆炸     | `d6!>4: [5, 3!] = 8`              |
| `/r d6!3`  | 掷出 =3 即爆炸     | `d6!3: [3, 5!] = 8`               |
| `/r 5d6!!` | 复合爆炸（追加值合并）   | `5d6!!: [2, 3, 5, 8, 14] = 32`    |
| `/r 5d6!p` | 穿透爆炸（追加骰 -1）  | `5d6!p: [3, 5, 2, 6, 1, 1!] = 18` |

### 成功 / 失败计数

骰子不再求和，而是统计满足条件的骰数。

| 输入           | 说明          | 输出示例                               |
|--------------|-------------|------------------------------------|
| `/r 3d6>3`   | ≥3 算成功      | `3d6>3: [1, 4*, 5*] = 2成功`         |
| `/r 10d6<4`  | ≤4 算成功      | `10d6<4: [2*, 3*, 5, 6, 1*] = 3成功` |
| `/r 3d6>3f1` | ≥3 成功，=1 失败 | `3d6>3f1: [4*, 1x, 6*] = 2成功 1失败`  |

> `*` 标注成功骰，`x` 标注失败骰。

### 重骰

| 输入             | 说明           | 输出示例                                  |
|----------------|--------------|---------------------------------------|
| `/r 2d8r<2`    | ≤2 循环重掷      | `2d8r<2: [~1~, 7, 5] → [7, 5]`        |
| `/r 8d6r`      | =1 循环重掷（默认）  | `8d6r: [2, 3, 5, 4, 6, 3, 6, 4] = 33` |
| `/r 2d6ro<2`   | ≤2 只重掷一次     | `2d6ro<2: [~1~, 4, 6] → [4, 6]`       |
| `/r 2d8r1r3r5` | =1、=3、=5 均重掷 | —                                     |

> 被重骰替换的原始值显示为 `~N~`。

### 排序

| 输入         | 说明   | 输出示例                                   |
|------------|------|----------------------------------------|
| `/r 8d6s`  | 升序显示 | `8d6s: [1, 2, 2, 3, 4, 5, 6, 6] = 29`  |
| `/r 8d6sd` | 降序显示 | `8d6sd: [6, 6, 5, 4, 3, 2, 2, 1] = 29` |

### DC 技能检定

在表达式后附加 `标签 DC值`（**标签与 DC 数字之间需有空格**），插件将自动比较掷骰总计与 DC：

| 输入                | 输出示例                                 |
|-------------------|--------------------------------------|
| `/r d20 感知 15`    | `感知 d20: [12] = 12 / 15 失败`          |
| `/r d20+3 奥秘 12`  | `奥秘 d20+3: [10] +3 = 13 / 12 成功`     |
| `/r d20adv 察觉 13` | `察觉 2d20adv: [(7), 16] = 16 / 13 成功` |

判定规则：

- 总计 >= DC → 成功
- 总计 < DC → 失败
- 天然 20 → 大成功（无论 DC，仅在单组 d20 表达式时生效）
- 天然 1 → 大失败（无论 DC，仅在单组 d20 表达式时生效）

## 配置项

在 AstrBot 管理面板的插件配置中可调整以下参数：

### 基础配置

| 配置项                   | 类型 | 默认值  | 说明                                   |

| `show_detail`         | 布尔 | true | 是否显示各骰子点数明细                          |

### 骰面大小配置

| 配置项                              | 类型 | 默认值   | 说明                                                                           |
|----------------------------------|----|-------|------------------------------------------------------------------------------|
| `default_dice_sides`             | 整数 | 20    | 全局默认骰面数，无参数执行 `/r` 时使用                                                       |
| `allow_session_dice_sides`       | 布尔 | true  | 是否允许用户通过 `/dset` 命令为当前会话设置独立骰面数                                              |
| `enable_whitelist`               | 布尔 | false | 是否启用白名单模式，启用后仅白名单用户（或管理员）可使用 `/dset` 和 `/rprefix`；**不影响 `/r` 掷骰，掷骰始终对所有人开放** |
| `whitelist_users`                | 列表 | []    | 允许使用 `/dset`、`/rprefix` 的用户 ID 列表，**留空则自动使用 AstrBot 管理员 ID**                 |

| `allow_custom_prefix` | 布尔  | true | 是否允许用户通过 `/rprefix` 命令为当前会话设置独立触发前缀                        |

| 配置项                  | 类型 | 默认値  | 说明                                  |
|----------------------|----|------|-------------------------------------|
| `enable_history`     | 布尔 | true | 是否开启投掷历史记录功能，开启后每次成功投掷将被记录          |
| `allow_view_history` | 布尔 | true | 是否允许用户通过 `/rh` 查看历史记录               |
| `max_history_count`  | 整数 | 50   | 每个会话最多保留的历史记录数，超出后自动删除最旧的记录（5–5000） |

## LLM 函数工具

插件注册了 `roll_dice` 工具，启用 Agent 模式后，LLM 可在 TRPG 叙事中主动调用掷骰，结果将同时发送给用户并返回给 LLM 用于继续叙事。

LLM 调用参数：

| 参数           | 类型     | 说明                                                                |
|--------------|--------|-------------------------------------------------------------------|
| `expression` | string | 骰池表达式，支持所有上述语法，如 `"4d6kh3"`、`"3d6>3f1"`、`"5d6!!"`                 |
| `label`      | string | 投掷说明，可附 DC 值（**标签与 DC 之间需有空格**），如 `"感知 15"`、`"力量豁免 12"`；不需要时传空字符串 |

## 与 Roll20 规范的差距

以下功能在 [Roll20 骰子规范](https://help.roll20.net/hc/en-us/articles/360037773133-Dice-Reference) 中明确定义，但当前版本
**尚未实现**：

| 未实现功能            | 规范语法示例                                    | 说明                                       |
|------------------|-------------------------------------------|------------------------------------------|
| 算式骰数 / 骰面        | `(2+3)d6`、`3d(2*4)`                       | 括号内表达式作为骰子数或面数                           |
| 完整数学运算符          | `2d6*5`、`d20%2`、`2**3`                    | 当前仅支持 `+`/`-` 连接各骰池，不支持 `*`、`/`、`%`、`**` |
| 数学函数             | `floor(2d6/2)`、`ceil()`、`round()`、`abs()` | 完全未实现                                    |
| 括号运算优先级          | `(2d6+3)*4`                               | 不支持括号改变运算顺序                              |
| 分组骰              | `{3d6+5, 2d8+4}kh1`                       | 花括号跨骰池保留 / 丢弃，每个子组分别求和后再筛选               |
| 自定义临界点 `cs`/`cf` | `1d20cs>18`、`1d20cf<3`                    | 当前天然 20 / 天然 1 硬编码于 d20，无法自定义临界范围        |
| 每组独立内联标签         | `2d6[Fire]+3d8[Ice]`                      | 当前仅支持全局单一标签，不支持为每个骰组附加独立说明               |
| 骰子匹配 `m`/`mt`    | `6d6mt3`、`6d6mt3>4`                       | 配对数 / 配对计数修饰符                            |
