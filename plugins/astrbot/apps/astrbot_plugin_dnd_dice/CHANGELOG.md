# Changelog

所有重要的更改都会记录在此文件中。

格式基于 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.0.0/)，
并且遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

## [0.6.1] - 2026-03-01

### 修复
- 修复 `/rh clear` 命令在 `enable_whitelist=False` 时群聊内任意用户均可清空历史的权限漏洞：新增独立的 `_check_history_clear_permission()` 逻辑，关闭白名单时群聊回退到 AstrBot 管理员判断，而非全放行。
- 修复 `RollHistoryManager.add()` 中读-改-写非原子导致并发投掷时历史条目互相覆盖的问题：引入按会话 key 的 `asyncio.Lock`，保证同一会话的写操作串行执行。
- 修复昵称、投掷表达式及结果中包含换行符等控制字符时可伪造历史行的文本注入风险：新增 `_sanitize()` 函数，在写入存储前和展示时双重剔除 `\r`、`\n`、`\t` 及其他非打印控制字符。

## [0.6.0] - 2026-03-01

### 新增
- 新增会话级投掷历史记录功能（`history.py` 模块）：每次投掷结果自动保存，支持按会话独立存储，可记录投掷表达式、结果摘要、发送者昵称与时间戳。
- 新增 `/rh` / `/rhistory` 命令：查看当前会话的投掷历史，群聊默认展示所有人的记录（含昵称），私聊仅展示自己的记录。
- 新增 `/rh me` 子命令：在群聊中筛选并展示当前用户自己的投掷历史。
- 新增 `/rh clear` 子命令：清除当前会话的全部历史记录，群聊中仅限白名单用户操作。
- 自定义触发前缀同步支持 `{prefix}rh` / `{prefix}rhistory` 命令。
- 新增 `enable_history` 配置项（默认 `true`）：全局启用或关闭投掷历史记录功能。
- 新增 `allow_view_history` 配置项（默认 `true`）：控制是否允许用户查看历史记录。
- 新增 `max_history_count` 配置项（默认 50，范围 5–500）：限制每个会话的最大历史条数，超出后自动淘汰最旧记录。

### 修复
- 修正 `metadata.yaml` 及 `README.md` 中插件描述里"掷骰"到"骰子"的术语不一致问题。

### 文档
- 更新 README，新增 `/rh` 系列命令说明、"投掷历史记录"功能章节及新增配置项文档。

## [0.5.1] - 2026-03-01

### 修复
- 修复 `CharacterSheet.__post_init__` 中 `skill_proficiencies` / `save_proficiencies` 未对 `None` 输入做兜底的问题；外部传入 `None`（插件生态中常见）时不再触发 `TypeError`，改为静默视为空集合。
- 修复 `_strip_label` 中 `Sm`/`Sc`/`Sk` 类 Unicode 符号（如数学运算符 `×`、`÷`、货币符号）会无条件截断表达式的问题；新策略：`L*` 自然语言文字与 `So`（Emoji）无条件截断为标签起点，`Sm`/`Sc`/`Sk` 仅在前有空白时才截断，避免粘贴进来的 Unicode 运算符被静默丢弃后给出难以理解的解析错误。
- 修复 `_apply_rerolls` 中多重重骰条件处理不完整的问题：移除首条命中后立即 `break` 的逻辑，改为按条件链顺序继续对更新后的骰值依次检查后续条件，实现完整多重重骰修饰语义。
- 修复非 `once` 重骰在深度耗尽后静默保留"仍命中条件"的值、缺乏可解释性的问题：深度耗尽且最终值仍落在应重骰区间时，骰子状态标记为 `kept_capped`，格式化层在骰值后追加 `?` 后缀（如 `3?`）提示用户该值未能成功重出。

## [0.5.0] - 2026-03-01

### 新增
- 新增 `max_input_len` 配置项：单次掷骰表达式允许的最大字符数，可在管理面板通过滑块调节（10–1000，默认 200），替代原先硬编码的 200 字符上限。
- 新增全角字符自动归一化（`_normalize_fullwidth`）：用户输入的全角运算符（如 ＋、－、（、））自动转换为对应 ASCII 字符，不再被误识别为标签分隔触发点。
- 新增净成功/失败计数格式化展示：成功计数模式下输出同时显示成功数、失败数及净值（如 `3成功 1失败 净+2`）。
- 新增爆炸追加骰与保留骰在成功计数模式下的 `*`/`x` 标注，展示更直观。
- 新增 `max_total_rolled` 参数限制单次掷骰实际产生的骰子总数上限（默认 500），防止多骰池加高爆炸深度组合生成超大列表。
- 新增前缀缓存 TTL 机制（300 秒）：管理面板修改全局或会话前缀后，缓存在 TTL 到期时自动刷新，无需重启插件。
- 新增有界 LRU 缓存用于会话级自定义触发前缀（上限 512 条）：提取专用 `_set_prefix_cache` 方法，驱逐逻辑更清晰。
- 新增有界 LRU 缓存用于角色卡内存管理，防止长时间运行时角色卡数据无限堆积。

### 修复
- 修复骰子面数/数量超出上限时错误信息表述不清晰的问题，提示中现明确显示实际值与限制值。
- 修复重骰历史记录追踪不完整的问题：补全中间重骰值的完整链路记录；保留骰组按值排序输出。
- 修复骰子表达式解析中不可识别修饰符（如拼写错误的关键词）未给出明确报错的问题，新增边界守卫防止解析器进入无限回退循环。
- 修复标签分离逻辑（`_strip_label`）中含运算符的纯 ASCII 表达式（如 `2d6 + 1d4 伤害`）被空格误切的问题：空格后紧跟 `+`/`-` 时整体去空格视为算式，再从下一个非算式位置切分标签。
- 修复非 ASCII 字符判断逻辑，改进表达式与标签的边界识别。
- 修复权限检查和输入校验逻辑，`/dset`、`/rprefix` 等命令对边界条件（空输入、非法符号等）处理更健壮。
- 修复 `/rprefix "/"` 拒绝逻辑：将 `/` 设为前缀会与系统默认前缀重叠，现予以明确拒绝。

### 重构
- 重构 `_annotate_die` 标注逻辑，引入按位置精确标注的 `die_rolls` 列表，删除约 45 行遗留频率计数回退代码。
- 重构 `is_natural_20`/`is_natural_1` 检测为共享方法 `_check_natural_roll`，消除重复判断逻辑。
- 重构 `formatter.py` 比较函数：将私有 `_compare` 替换为已公开的 `_cmp`，统一比较逻辑入口。
- 将 `_MAX_INPUT_LEN` 从函数内局部变量提升为模块级常量，便于配置层引用和覆盖。
- `CharacterSheet` 属性分值 Clamp 及等级限制改用显式 `_clamp()` 函数调用，替代原 `getattr`/`setattr` 循环；`CharacterManager` 加入 `asyncio.Lock` 保证并发安全。
- 全面将代码注释翻译为中文，提升可读性与维护一致性。

### 文档
- 更新 README，补充白名单字段说明及标签格式化规则说明。

## [0.4.0] - 2026-02-25

### 新增
- 新增自定义指令触发前缀支持：支持在管理面板设置全局默认前缀，或通过 `/rprefix` 命令为当前会话独立设置触发前缀（如 `.`、`!` 等），与系统默认前缀并存，不互相替换。
- 新增 `/rprefix` 命令：支持查询、设置、重置当前会话的自定义触发前缀；设置后 `{prefix}r`、`{prefix}roll`、`{prefix}dset` 等均可触发对应功能。
- 新增 `default_cmd_prefix` 配置项：全局默认额外触发前缀，留空则仅响应系统默认前缀。
- 新增 `allow_custom_prefix` 配置项：控制是否允许用户通过 `/rprefix` 命令设置会话级触发前缀。
- 自定义前缀与 `/dset` 共用同一套白名单配置（`enable_whitelist`、`whitelist_users`、`allow_private_bypass_whitelist`）。

### 文档
- 更新 README，补充 `/rprefix` 命令说明及自定义触发前缀相关配置项文档。

## [0.3.0] - 2026-02-25

### 新增
- 新增 `/dset` / `/dice_set` 命令：支持为当前会话（群聊或私聊）独立设置默认骰面数，设置后 `/r` 无参数时将投掷指定面数的骰子。
- 新增 `default_dice_sides` 配置项：全局默认骰面数（初始为 20），无参数执行 `/r` 时使用。
- 新增 `allow_session_dice_sides` 配置项：设为 `false` 后全局禁用 `/dset` 命令。
- 新增 `enable_whitelist`、`whitelist_users`、`allow_private_bypass_whitelist` 配置项：对 `/dset` 命令进行白名单权限控制，私聊可配置为绕过限制。

### 修复
- 修复 `/dset` 无参数查询时，响应中默认骰面数来源（会话设置 / 全局默认）描述不清晰的问题。

### 文档
- 更新 README，新增骰面大小设置与权限控制说明。

## [0.2.1] - 2026-02-25

### 新增
- 新增配置值安全转换函数 `_safe_int` 与 `_safe_bool`，防止管理面板填入非法值时导致插件加载失败。
- 新增角色卡管理器（`CharacterManager`）懒加载机制，避免核心持久化接口未实现时被意外调用。
- 新增 MIT 许可证文件。

### 修复
- 修复 `d20dis`（劣势骰语法糖）解析异常的问题：`dis` 后缀现可被正确识别为 `kl1` 简写。
- 修复骰池表达式解析中的边界处理逻辑，新增输入长度限制（最长 200 字符），防止超长表达式造成性能问题。
- 修复插件在 AstrBot 插件市场中显示名称与描述不准确的问题。

### 优化
- 重构掷骰执行引擎，完善内部文档注释，提升代码可读性与可维护性。
- 修正修正值（modifier）计算逻辑，修复负修正值运算异常。
- 角色卡 `AbilityScores` 属性分值 Clamp 至 [1, 30]，`CharacterSheet` 等级 Clamp 至 [1, 20]，规范化熟练项集合类型。

### 文档
- 更新 README，改进指令格式说明与排版清晰度。

## [0.2.0] - 2026-02-25

### 新增
- 初始版本发布。
- 支持完整 Roll20 骰池语法：基础骰（`d20`、`2d6+5`）、FATE/Fudge 骰（`4dF`）、保留最高/最低（`4d6kh3`、`2d20kl1`）、丢弃最高/最低（`8d6d3`、`8d6dh3`）、优势/劣势（`d20adv`、`d20dis`）。
- 支持三种爆炸变体：标准爆炸（`d6!`）、复合爆炸（`5d6!!`，Shadowrun 风格）、穿透爆炸（`5d6!p`，HackMaster 风格）；支持自定义爆炸阈值（`d6!>4`）。
- 支持目标数成功计数（`3d6>3`）与失败计数（`3d6>3f1`）。
- 支持重骰：循环重骰（`2d8r<2`）与单次重骰（`2d6ro<2`）。
- 支持结果排序：升序（`8d6s`）与降序（`8d6sd`）。
- 支持多骰组合与修正值（`2d6+1d4+3`）。
- 支持标签注释（`d20 攻击检定`、`d20#攻击检定`）与 DC 技能检定判定，天然 20 强制大成功、天然 1 强制大失败。
- 注册 LLM 函数工具 `roll_dice`，Agent 模式下 LLM 可在 TRPG 叙事中主动发起掷骰。

### 优化
- 重构修正值计算逻辑，修复负修正值运算问题。
